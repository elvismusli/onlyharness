# Consent boundaries

Routing consent covers sending only the displayed sanitized task summary to SuperSkill. Activation consent is separate and follows exact release, digest, named checks, permissions, limitations, client and temporary/pinned disclosure. Keep and remove each require their own explicit confirmation.

Public browse-catalog install consent is another separate boundary. Before installing an unreviewed hosted skill, disclose its exact ID/version, scan/risk/license state, native target and lack of managed approval. Require direct acknowledgement before using `--allow-unreviewed`. Never treat browse install as activation, pinning or managed review. Never auto-focus or imply a high-risk confirmation.
